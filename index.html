HTML Forms

Form Tags

Tag Name Attributs Values Description Tip
<form> Form action, method, autocomplete, target, name, rel, enctype, accept-charset, novalidate The wrapper element for all different types of input elements.
action <form action="/action_page.php"> Defines the action to be performed when the form is submitted. Usually, the form data is sent to a file on the server when the user clicks on the submit button. If the action attribute is omitted, the action is set to the current page.
method <form action="/action_page.php" method="get">

<form action="/action_page.php" method="post"> 		Specifies the HTTP method to be used when submitting the form data. The form-data can be sent as URL variables (with method="get") or as HTTP post transaction (with method="post"). The default HTTP method when submitting form data is GET. NEVER use GET to send sensitive data!
autocomplete <form action="/action_page.php" autocomplete="on"> Specifies whether a form should have autocomplete on or off. When autocomplete is on, the browser automatically complete values based on values that the user has entered before.
target <form action="/action_page.php" target="_self"> Specifies where to display the response that is received after submitting the form. The default value is _self which means that the response will open in the current window.
target="_blank" The response is displayed in a new window or tab.
target="_parent" The response is displayed in the parent frame.
target="_top" The response is displayed in the full body of the window.
target="framename" The response is displayed in a named iframe.

<input> Input type button, checkbox, placeholder, autofocus, autocomplete, high, width, list, color, date, datetime-local, email, file, hidden, image, month, number, password, radio, range, reset, search, submit, tel, text, time, url, week The input element can be displayed in several ways, depending on the type attribute.
<input type="text"> Defines a single-line text input field.
<input type="password"> Defines a single-line text input field.
<input type="submit"> Defines a button for submitting form data to a form-handler. The form-handler is typically a server page with a script for processing input data. The form-handler is specified in the form's action attribute
<input type="reset"> Defines a reset button that will reset all form values to their default values.
<input type="radio"> Radio buttons let a user select ONLY ONE of a limited number of choices:.
<input type="checkbox"> Checkboxes let a user select ZERO or MORE options of a limited number of choices.
<input type="button"> Defines a button.
<input type="email"> The Input Type Email is used for input fields that should contain an e-mail address. Depending on browser support, the e-mail address can be automatically validated when submitted.
<input type="image"> Defines an image as a submit button. The path to the image is specified in the src attribute. <input type="image" src="img_submit.gif" alt="Submit" width="48" height="48">
<input type="file"> Defines a file-select field and a "Browse" button for file uploads.
<input type="number"> Defines a numeric input field. I can also set restrictions on what numbers are accepted.
<input type="range"> defines a control for entering a number whose exact value is not important (like a slider control). Default range is 0 to 100. However, you can set restrictions on what numbers are accepted with the min, max, and step attributes.
checked Specifies that an input field should be pre-selected when the page loads. For type="checkbox" or type="radio".
required Specifies that an input field is required. Must be filled out.
value Specifies the default value for an input field.
size Specifies the width (in characters) of an input field.
<label> Label for <label for="..."> Defines a label for several form elements. The for attribute of the label tag should be equal to the id attribute of the input element to bind them together. The label element is useful for screen-reader users. The label element also help users who have difficulty clicking on very small regions (such as radio buttons) - because when the user clicks the text within the label element, it toggles the radio button/checkbox.
<select>
<option> 		Drop-down list id, name, size, value, selected, multiple <select id="..." name="..." size="2" multiple>
<option value="1">1</option>
<option value="2" selected >2</option>
</select> 		The select element defines a drop-down list. The option element defines an option that can be selected. To define a pre-selected option, add the selected attribute to the option. By default, the first item in the drop-down list is selected. Use the size attribute to specify the number of visible values. Use the multiple attribute to allow the user to select more than one value.
<textarea> Textarea name, rows, cols <textarea name="..." rows="2" cols="5">
Multi-line input field </textarea> 		The element defines a multi-line input field (a text area).
<button> Button type, onclick <button type="button" onclick="alert('Hello')">Click Me!</button> The element defines a clickable button.

</datalist> 		The element specifies a list of pre-defined options for an input element. Users will see a drop-down list of the pre-defined options as they input data. The list attribute of the input element, must refer to the id attribute of the datalist element.
<optgroup> Option-group global, event, disabled, label Defines a group of related options in a drop-down list. If you have a long list of options, groups of related options are easier to handle for a user.
